为什么这样建

先把框架固定，后面就只维护数据。

你这类项目最怕一开始技术栈太重，后期维护成本反而压垮更新频率。纯 HTML + Tailwind CSS + Cloudflare Pages 的优势是稳定、便宜、部署快，而且以后即便你只会改 JSON，也能继续维护。

众包模式

网站会越来越全，数据更新节奏才是核心。

以后你主要维护两件事：一是医生和诊所的 JSON，二是 Google Forms 收集来的用户提交。结构固定后，新增数据会自然累积，不需要频繁推倒重来。
